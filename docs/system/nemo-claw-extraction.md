## C. The Triple-Engine Extraction Logic
### OpenCLAW
Reason about required interaction evidence and fail-safe route selection.

### Goose
Extract required UI interactions as deterministic step sequences with pre/post checks.

### NemoClaw
Execute route navigation, element anchoring, form interaction, and state verification across app surfaces.

## D. Hybrid Data Lifecycle
### SQLite (High-Performance)
Store anchor maps, interaction traces, selector confidence, and replay metadata.

### Vault (Secure Commit State)
Commit critical interaction evidence and approved anchor revisions.

## E. Channel and Execution
- Cronjobs: Daily selector drift and broken-anchor audits.
- Internal Chat: UI automation reliability channel.
- External Channels: Telegram for repeated automation failure alerts.
- Dynamic UI Contract: System and module screens must be schema-driven by registry YAML/JSON definitions.
- No-Dead-End Navigation: Every screen must expose Back and Home controls for Electron no-URL execution.
- Manual Override: Show Current State and Proposed Improvement before every registry-impacting commit.
